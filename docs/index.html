Feature list


    Who What Where
  • See the IP addresses that your computer communicates with, sorted by amount of packets transmitted. Shows GeoIP and a ping result for each address. Can be used for determining a game server IP or monitoring suspicious internet connections.

    • Ping to Speech

  • Press a hotkey and hear the result of a ping sent to a predefined IP address or hostname. Useful while playing online games.

    • Watchdog

  • Inspects network traffic and waits for a packet from a predefined list of IP addresses and source/destination ports and several protocols. Upon finding a match, a predefined message will be read out by text-to-speech. This feature can be used as a notifier for an incoming connection, for example.

    • Visual Trace

  • Perform a trace-route for a given IP address or hostname. Once the trace is complete, a visual trace is available, which shows the route of your packet on a map.


Installating and Running


Prerequisites

  • Java Runtime Environment (JRE) 1.8_60 or newer

    Download JRE here. If you already have JRE installed and want to check the version, open the command line (cmd) and type "java -version".

  • WinPcap

    Download WinPcap here. WinPcap is a program that allows capturing network packets on Windows.


Running the program
Once you've installed Java and WinPcap, download WhoWhatWhere and extract the archive. Run the executable and allow ~10 seconds to load. This program requires administator rights in order to capture network packets. If you're prompted by Windows, allow it access.


How to Use


Who What Where can be minimized to the systrem tray by pressing the 'x' (close window) button. In order to exit for the program, press the Exit button or use the menu bar "File".

Pick your netowrk adapter:

In the top-left pane there should be a list with at least one network adapter (example). Select the adapter your want to listen to. If you're not sure which one to choose (the name would often have "ethernet" or "wireless" in it), just try and see which one works.

Who What Where

 Who What Where listens to (aka "captures") your internet traffic and shows you which IP addresses were contacted, as well as some additional info such as ping time and geographical location of the IP address. Set your preferred settings, start and stop a capture - and the results will be shown in the table.
Capture Options
  • Protocol selection: If no protocol is selected, then all protocols will be counted. Otherwise, only packets of the selected protocols will be counted.
  • Automatically/manually stop capture: Choose if you'd like to stop the capture yourself, or let it stop automatically after a certain amount of time.
  • Capture options:
    Get location info for each IP address - If checked, every IP address entry will also show its geographical location information (see limitations). If unchecked, the results will be displayed a bit quicker.
    Ping each IP address - If checked, each IP address entry will also show the ping result to that IP. There is also a configurable timeout. If unchecked, the results will be displayed a bit quicker.

Right-clicking on a result in the table opens a pop-up menu with several options: "See more GeoIP results in the browser" loads a GeoIP service results in the default web-browser. This site might have more (or different) results than what's shown in the table. See GeoIP Limitations.

Hotkey and Text-To-Speech Options
Instead of manually pressing the start/stop buttons, you can use a hotkey. The hotkey is active globally, meaning you can press it while using any other application and even when Who What Where is minimized to the system tray. If a capture is running, the hotkey will stop it. Otherwise, it will start a new capture.
If the text-to-speech feature is enabled and you use the hotkey to start or stop the capture, the results will be read out ot you - so you don't have the look at the application screen. This is useful when you already know what you're looking for.
You can also filter out what is read to you, the first X rows or only rows that contain a certain value in a certain column. You can also select which column values to read out.

Ping to Speech

 Put in any IP or hostname, and press the hotkey. The ping result will be read out.
The hotkey is configurable, and the default one (Ctrl+NumPad 0) requires the numlock on your keyboard to be on.

Watchdog

 Watchdog listens to your network traffic and informs you when a packet arrives from a certain IP/port/protocol.
Click "Manage match list" to add entries to the list, or load an existing preset. After clicking "Done", write a message to read out when a match from the list is found. You can preview the message with the preview button.
Once the list and message are set, click "Start waiting for match" or press the hotkey. The default hotkey (Ctrl+NumPad 1) requires the numlock on your keyboard to be on.

Visual Trace

 Put in any IP or hostname, and click "Trace". A trace window opens, where you can configure a ping timeout and whether or not to resolve hostnames.
Click "Start trace" and wait for the trace to complete. Then click "Show visual trace".
In the Visual Trace screen you can check/uncheck any hop on the route and see the changes on the map. You can also click "Focus map here" to see the exact location of the hop zoomed in, or you can click "GeoIP Info" to view info from another GeoIP source on your default browser.

Note about GeoIP Limitations:

Geographical IP services aren't always accurate. The address is based on what the company that owns the address reported when the address was bought. Sometimes an international company might list all their IPs are located in one area (where the main offices are, for exmaple), but some of these IPs may be used in other areas that the company operates. This is why sometimes for one IP address there may be more than one possible location, or an incorrect one.
